# 1. main (시작점)

👉 아무 생각 없이 **ui만 실행**

```
render_home()
render_sidebar()
```

✔ 역할

- 앱 시작
- 어떤 UI 보여줄지 결정

👉 **비즈니스 로직 없음**

---

# 2. ui (화면)

👉 **유저랑 직접 만나는 곳**

```
st.title()
st.selectbox()
st.dataframe()
```

✔ 하는 일

- 입력 받기
- service 호출
- 결과 화면에 출력

👉 느낌:

```
data=service.get_data()
st.dataframe(data)
```

👉 **“그리기 + 연결만 한다”**

---

# 3. service (중간 처리)

👉 **데이터를 가져와서 가공하는 곳**

✔ 하는 일

- db 호출
- DataFrame 변환
- 계산/가공

```
columns,rows=client.select(query)
df=pd.DataFrame(rows,columns=columns)
```

👉 **“데이터를 사람이 쓰기 좋게 만든다”**

---

# 4. db_client (DB 연결)

👉 이미 잘 만들어둔 핵심 부분 👍

✔ 하는 일

- DB 연결 관리
- 쿼리 실행
- rows / columns 반환

👉 느낌:

```
columns,rows=client.select(query)
```

👉 **“DB랑 직접 대화하는 애”**

---

# 전체 흐름 예시

```
[사용자]
   ↓
ui (버튼 클릭)
   ↓
service (데이터 요청)
   ↓
db_client (쿼리 실행)
   ↓
DB
   ↓
db_client (결과 반환)
   ↓
service (DataFrame + 가공)
   ↓
ui (st.dataframe 출력)
```

---

# 한 줄씩 요약

- **main**: 시작만 한다
- **ui**: 그리고 보여준다
- **service**: 데이터 가공한다
- **db_client**: DB랑 통신한다
- **queries**: SQL 모아둔다
